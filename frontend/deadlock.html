Understanding Deadlocks

üí° What is a Deadlock?

A deadlock is a situation in computing where two or more processes are unable to proceed because each is waiting for the other to release a resource, creating a cyclic dependency that halts system progress.

üìå The Four Necessary Conditions

Mutual Exclusion

Resources cannot be shared; only one process can use a resource at a time.

Hold and Wait

Processes hold resources while waiting for additional ones.

No Preemption

Resources cannot be forcibly removed from processes.

Circular Wait

A circular chain of processes exists where each waits for a resource held by the next.

üîç Deadlock Detection


Deadlocks can be detected using graph-based techniques:

  • Resource Allocation Graph (RAG): Represents processes and resources with edges indicating allocation and requests.
  • Wait-For Graph (WFG): A simplified RAG that only tracks dependencies among processes.
  • Cycle Detection Algorithms: Depth-First Search (DFS) or algorithms like Banker's Algorithm can be used to identify circular dependencies.

üõ†Ô∏è Deadlock Handling Strategies

  • Prevention: Design system to negate one of the four conditions.
  • Avoidance: Use algorithms (like Banker's) to ensure safe states.
  • Detection & Recovery: Allow deadlocks but detect and recover from them.
  • Ignorance: Pretend deadlocks never occur (used by many OS).
