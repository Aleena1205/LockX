What is a Wait-For Graph?

A Wait-For Graph (WFG) is a directed graph used to detect deadlocks in operating systems:

  • Nodes: Represent processes only
  • Edges: Directed edges show dependencies (P1 â†’ P2 means P1 is waiting for P2)
  • Cycles: Any cycle in the graph indicates a deadlock situation

How Deadlock Detection Works

1

The system monitors process dependencies and builds the WFG

2

Regularly checks for cycles in the graph using algorithms like Depth-First Search

3

If a cycle is found, the system can take corrective action (kill process, rollback, etc.)

Real-world Deadlock Examples

Database Systems

Transactions waiting for locks on database rows can deadlock when they request resources in different orders.

Network Communication

Servers waiting for responses from each other can deadlock if both are blocked waiting for the other.

Printer Allocation

Processes holding different printers while waiting for others can create circular dependencies.
