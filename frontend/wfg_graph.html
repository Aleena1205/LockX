What is a Wait-For Graph?

A Wait-For Graph (WFG) is a directed graph used to detect deadlocks in operating systems. It shows the dependencies between processes and resources:

  • Nodes: Represent processes (circles) and resources (squares)
  • Edges: Show relationships (holds/waiting between processes and resources)
  • Cycles: A cycle in the graph indicates a deadlock situation

How Deadlock Detection Works

The system detects deadlocks by searching for cycles in the WFG:

1

Build the graph showing all process-resource dependencies

2

Perform cycle detection using algorithms like Depth-First Search

3

If a cycle is found, processes in the cycle are deadlocked

Real-world Deadlock Examples

Database Systems

Transactions waiting for locks on database rows can deadlock when they request resources in different orders.

Network Communication

Servers waiting for responses from each other can deadlock if both are blocked waiting for the other.

Printer Allocation

Processes holding different printers while waiting for others can create circular dependencies.
