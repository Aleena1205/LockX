What is Deadlock?

A deadlock is a situation where a set of processes become permanently blocked because each process is waiting for a resource that another process is holding. This prevents further execution of the processes, leading to a system halt if not resolved.

ðŸ“Œ Four Necessary Conditions for Deadlock

For a deadlock to occur, the following four conditions must hold simultaneously:

  • Mutual Exclusion: Only one process can use a resource at a time. If another process requests the resource, it must wait until the resource is released.
  • Hold and Wait: A process holding at least one resource is waiting to acquire additional resources held by other processes.
  • No Preemption: A resource cannot be forcibly removed from a process; it must be released voluntarily.
  • Circular Wait: A set of processes form a circular chain where each process is waiting for a resource that the next process in the chain holds.

If a cycle is detected in the Wait-For Graph (WFG), a deadlock is confirmed.

ðŸ›‘ How Deadlocks Occur

Deadlocks often occur in multi-processing systems, databases, and distributed systems where multiple processes share resources. Common scenarios include:

  • Two processes each holding a resource and waiting for the other to release its resource.
  • Multiple transactions in a database trying to lock the same records.
  • Threads in a multi-threaded program waiting for shared variables or memory.

ðŸ”Ž Deadlock Detection

Deadlocks can be detected using graph-based techniques:

  • Resource Allocation Graph (RAG): Represents processes and resources with edges indicating allocation and requests.
  • Wait-For Graph (WFG): A simplified RAG that only tracks dependencies among processes.
  • Cycle Detection Algorithms: Depth-First Search (DFS) or algorithms like Banker's Algorithm can be used to identify circular dependencies.

âœ… Deadlock Prevention Techniques

  • Resource Hierarchies: Assigning a fixed order to resource allocation to prevent circular wait.
  • Request-All-At-Once: Ensuring a process requests all needed resources at the start to avoid hold and wait.
  • Timeouts: Setting time limits on resource holding to break potential deadlocks.
  • Process Prioritization: Allowing high-priority processes to preempt resources from lower-priority processes.

ðŸ”„ Deadlock Recovery Strategies

If a deadlock is detected, the system can recover using one of the following methods:

  • Process Termination: Killing one or more processes to break the circular dependency.
  • Resource Preemption: Forcing processes to release resources and restarting them later.
  • Rollback: Reverting processes to a previous safe state.
